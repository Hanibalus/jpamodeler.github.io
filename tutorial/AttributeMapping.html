JPA Modeler supports the following mapping types for Java persistent entities:

ID Mapping :

Use an ID Mapping to specify the primary key of an entity. ID mappings may be used with a Entity or Mapped superclass.
The primary key must be defined on the entity class that is the root of the entity hierarchy or on a mapped superclass that is a (direct or indirect) superclass of all entity classes in the entity hierarchy. The primary key must be defined exactly once in an entity hierarchy.
To create a ID mapping:
  • Click on the Entity , then click on the expand button > ID property.
  • Double click on the attribute to rename it.
  • To change field type, goto property panel > Field Type > Click to open Field Type panel : ID mappings may be used only with the following attribute types:
    • Java primitive types and wrappers of the primitive types
    • java.lang.String, java.math.BigInteger, java.math.BigDecimal
    • java.util.Date
    • java.sql.Date

Basic Mapping :

Use a Basic Mapping to map an attribute directly to a database column.
To create a basic mapping:
  • Click on the Entity , then click on the expand button > basic property.
  • Double click on the attribute to rename it.
  • To change field type, goto property panel > Field Type > Click to open Field Type panel : Basic mappings may be used only with the following attribute types:
    • Java primitive types and wrappers of the primitive types
    • java.lang.String, java.math.BigInteger, java.math.BigDecimal
    • java.util.Date, java.util.Calendar
    • java.sql.Date, java.sql.Time, java.sql.Timestamp
    • byte[], Byte[], char[], Character[]
    • enums
    • any other type that implements Serializable

Element collection mapping (Basic) :

Use an Element Collection to define a collection of Basic objects. The Basic values are stored in a separate collection table. Because the target is a Basic value (instead of an Entity), you can easily define collections of simple values without having to define a class for the value.
To create a Element Collection :
  • Click on the Entity , then click on the expand button > Basic Element Collection property.
  • Double click on the attribute to rename it.

Embedded mapping :

Use an Embedded Mapping to specify a persistent field or property of an entity whose value is an instance of an embeddable class.
To create a Embedded mapping :
  • Click on the Entity , then click on the composition button .
  • Now, drag embedded attribute to Embeddable class.
  • Double click on the attribute to rename it.
Please click here for more info : Embeddable

Version mapping :

Specifies the version field or property of an entity class that serves as its optimistic lock value. The version is used to ensure integrity when performing the merge operation and for optimistic concurrency control.
An entity is automatically enabled for optimistic locking if it has a property or field mapped with a Version mapping.
Only a single Version property or field should be used per class; applications that use more than one Version property or field will not be portable.
To create a Version mapping :
  • Click on the Entity , then click on the expand button > version property.
  • Double click on the attribute to rename it.
  • To change field type, goto property panel > Field Type > Click to open Field Type panel : Version mappings may be used only with the following attribute types:
    • int, java.lang.Integer
    • short, java.lang.Short
    • long, java.lang.Long
    • java.sql.Timestamp

Transient mapping :

Use the Transient Mapping to specify a field of the entity class that is not persistent.
It is used to annotate a property or field of an entity class, mapped superclass, or embeddable class.
To create a Version mapping :
  • Click on the Entity , then click on the expand button > transient property.
  • Double click on the attribute to rename it.
  • To change field type, goto property panel > Field Type > Click to open Field Type panel : Version mappings may be used only with the following attribute types:
    • int, java.lang.Integer
    • short, java.lang.Short
    • long, java.lang.Long
    • java.sql.Timestamp
